# EulerSolver_CFD_Training

A C++ finite-volume Euler solver training project for 2D compressible-flow simulations, with MPI parallelization and a modular solver structure for numerical-method development.

## Overview

This repository contains a training-oriented and research-oriented compressible Euler solver written in C++.

The code is designed to support progressive development of a CFD solver framework centered on the 2D Euler equations, finite-volume discretization on structured grids, MPI-based parallel execution, configurable numerical fluxes, reconstruction methods, time integrators, boundary conditions, and case-driven initialization through configuration files.

This project is intended for solver training, numerical-method testing, and future CFD research development.

## Current Capabilities

At the current stage, the solver supports the following features.

- 2D compressible Euler simulations
- Structured Cartesian grids
- MPI domain decomposition
- Configurable flux functions
- Configurable time integrators
- Configurable reconstruction methods
- Configurable initial conditions through setFields
- Case-based execution from plain-text configuration files

## Requirements

To build and run this project, you need the following environment.

- A C++ compiler with C++17 support
- An MPI implementation such as OpenMPI or MPICH
- macOS or Linux is recommended

The build script prefers mpicxx if available. If mpicxx is not found, it attempts to detect MPI include and library paths automatically.

## Build

Make the build script executable and run it.

chmod +x make.sh
./make.sh

By default, this builds an executable named solver.

The build script supports the following modes.

./make.sh
./make.sh release
./make.sh debug
./make.sh clean

To force a compiler, use the following command.

CXX=clang++ ./make.sh

To provide an MPI installation prefix, use the following command.

PREFIX=/path/to/mpi ./make.sh

## Run

The executable accepts a case configuration file as input.

./solver cases/case2d_sod.cfg

If no configuration file is provided, the program defaults to the following case.

cases/case2d_sod.cfg

### Serial Run

./solver cases/case2d_sod.cfg

### Parallel Run

mpirun -np 4 ./solver cases/case2d.cfg

When running in parallel, make sure that the following condition is satisfied.

mpi.px multiplied by mpi.py must equal the number of MPI processes

For example, if a case file contains the following settings,

mpi.px = 4
mpi.py = 2

then the run should use 8 MPI processes.

mpirun -np 8 ./solver cases/case2d.cfg

## Example Cases

The repository currently includes example configuration files in the cases directory.

- case2d.cfg      2D four-region Riemann-type problem
- case2d_sod.cfg  2D narrow-channel Sod-type test

These 2D case files define the following settings.

- Problem dimension
- Mesh resolution
- Computational domain
- Gas constant ratio gamma
- CFL number
- Final simulation time
- Numerical flux
- Time integrator
- Boundary condition
- MPI decomposition
- Reconstruction options
- Region-based initial fields
- Output control

## Configuration Style

The solver uses plain-text key-value configuration files.

Typical options include the following settings.

dim = 2
nx = 400
ny = 400
gamma = 1.4
cfl = 0.05
finalTime = 0.30
flux = hllc
timeIntegrator = rk4
bc = zeroGradient
mpi.px = 4
mpi.py = 2
outPrefix = solution
outputEvery = 100
writeFinal = true

Initial conditions can be defined through setFields, including background values and rectangular regions.

## Output

Simulation results are written using the configured output controls, such as output prefix, write interval, and final write option.

The code is structured so that flow-field output can be used for post-processing and visualization.

Because this is still a training solver and development solver, output format, diagnostics, and supported workflows may continue to evolve.

## Notes on Parallel Runs

MPI is initialized even for serial execution, so a single-process run still goes through MPI initialization.

For parallel cases, make sure of the following points.

- mpi.px multiplied by mpi.py matches the process count
- A compatible MPI launcher such as mpirun is used
- The MPI compiler and runtime environment are configured correctly on the target platform
